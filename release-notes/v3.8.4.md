# v3.8.4 — Gemma 4 E4B → 12B-4bit 교체 (분류 정확도 향상)

## Summary

MindVault 가 contradiction 4-way 분류·메모리 추출·alias 생성·query intent·검색 재순위에 쓰는
로컬 LLM 을 Gemma 4 E4B → **Gemma 4 12B-4bit** 로 교체했다. 추론 백엔드도 mlx-lm 서버 →
**mlx-vlm 서버**(전용 venv, 시스템 python 무오염)로 전환했다. 12B 가 instruction following
(특히 분류 정확도)에서 E4B 보다 우수해 contradiction 분류·extractor 신뢰도가 개선된다.
일상 bulk 변환(요약·번역·분류)은 요청 body 에 `enable_thinking=false` 를 넣어 reasoning 을 꺼
3~7배 가속한다(분류 품질은 동일).

## Changed

- src 모델 ID 를 운영 배포본과 동기화 — `memory_extractor` / `alias_generator` /
  `query_intent` / `search` / `contradiction_detector` 모듈의 Gemma 모델 ID 를 12B 로 (`2c2124d`)
- 추론 백엔드: mlx-lm → mlx-vlm (launchd 서비스, 포트 8080 동일, `--enable-thinking` 기동)
- 구 E4B + 테스트용 12B-8bit 모델 삭제로 디스크 ~17GB 확보, Gemma 단일화(12B-4bit)

## Notes

- Apple Silicon 32GB 에서 12B-4bit 약 8GB 로드
- 일상 bulk 변환은 `enable_thinking=false` 권장 (3~7배 가속, 분류 품질 동일)
- 테스트 카운트 변동 없음 (모델 ID 문자열 교체 — 로직 불변)
