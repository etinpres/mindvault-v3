# v3.8.3 — 전체 시스템 적대적 점검 R2 (결함 수정 + BGE-M3 잔재 정리)

## Summary

`/goal "전체 시스템 점검 → 버그 수정 → 추가 발견 없을 때까지 반복"` 으로 v3.8.2 코드베이스
전체(src 27모듈 + hooks + scripts + install/uninstall)를 **6라운드 적대적 워크플로**(find →
adversarial verify → runtime 재현)와 **2회 codex 독립 검증**(자기검증 confirmation bias 회피)으로
재점검했다. 776 테스트가 이미 그린이던 성숙한 코드에서 실재 결함을 찾아 수정하고 각 수정에
회귀테스트를 붙였다. 라운드별 발견 추세 **26 → 7 → 3 → 1 → 0(dry)** + codex 2(robustness) →
**R6 0(dry)** 로 수렴 확인. severity 도 라운드가 진행될수록 하락(R2 이후 발견은 대부분 직전
수정의 회귀/미완성 — 외부 결함은 R1 에서 소진).

추가로 유지보수자 지시에 따라 **BGE-M3 잔재를 전면 정리**했다(Sprint 9/14 Arctic-ko 교체 후
잔존하던 죽은 서버·plist·A/B 도구 삭제 + stale 함수명/주석 정정). reverify 의 BGE-stale **탐지**
기능은 보존(메모리 속 옛 모델명 언급을 잡는 안티-stale 장치이지 잔재가 아님).

테스트: **776 passed + 41 subtests → 815 passed + 41 subtests + 1 skipped** (회귀 0).

## Fixed — MED

- **Gemma 추출 JSON 끝쉼표 → 유효 후보 영구 negative-cache 유실** (`src/memory_extractor.py`):
  흔한 LLM 결함(끝쉼표)으로 strict `json.loads` 가 실패하면 유효 후보가 담긴 응답이 `[]` 로
  처리되고 negative cache 에 박혀 그 세션의 기억 후보가 영영 유실. `parse_failed` 신호로
  negative-cache 를 회피하고, **문자열 리터럴 밖의 구조적 끝쉼표만** 제거하는 string-aware
  스캐너로 복구(body 안 `[a, b,]` 같은 쉼표는 보존 — blind regex 의 콘텐츠 손상 회귀 방지).
  산문 머리말 + 끝쉼표 조합도 balanced-span 폴백으로 복구.
- **손상 vec 행 한 건이 전체 검색/매칭을 0건화** (`src/search.py`, `src/memory_search.py`,
  `src/memory_compiler.py`): 4의 배수가 아닌 손상 blob 은 shape 가드 도달 전 `np.frombuffer`
  ValueError, 차원 불일치 row 는 matmul ValueError 로 전파돼 단 한 행이 recall/dedup 매칭
  전체를 무력화. per-row try/except + 차원·finiteness 가드로 손상 행만 skip(NEXT-36 복원력 확장).
- **stale alias_index 의 ghost 결과 주입** (`src/memory_search.py`): 삭제·이동된 메모리 path 가
  alias_index 에 잔존하면 빈 name/desc 의 ghost 결과가 score 1.0 으로 정상 메모리보다 위로 주입.
  `memories` 테이블 실재 path 로 alias 후보 필터. + vec 서버 다운 시 `vec_available` 가 alias
  sentinel 로 오염돼 fts-only fallback 게이트 면제가 무력화되던 것을 합류 전 캡처로 교정.
- **`write_staged` 비-PID tmp → 동시 SessionEnd lost update** (`src/session_memory_end.py`):
  고정 `.tmp` 가 동시 same-slug SessionEnd 에서 cross-process unlink/replace race 로 후보 유실.
  PID-고유 tmp(reverify/alias_generator 와 동일 패턴).
- **uninstall 이 deploy_drift_check SessionStart 훅 잔재 미제거** (`uninstall.sh`): 파일 삭제 후
  존재하지 않는 경로를 가리키는 dangling SessionStart 훅이 잔존. settings 매칭 전용 substring
  배열을 rm 경로 배열과 분리(rm 루프가 cwd 동명 파일을 삭제하던 footgun 동시 차단).

## Method — 적대적 수렴 점검

- 6라운드 워크플로: finder 후보 → 독립 verifier 가 적대적 반증(코드 재확인 + 호출 도달성 +
  테스트 커버 + runtime 재현). 라운드 간 수정 코드를 재투입해 fix-the-fix 도 검출(R2~R4 가 R1
  수정의 회귀/미완성을 잡음).
- **codex 독립 검증 2회**: 큰 수정 직후 read-only 독립 리뷰로 자기검증 confirmation bias 회피.
  codex 가 R5 dry 이후에도 2건(robustness 가드 누락: aliases 컨테이너 타입·embed_cache TypeError)
  을 추가 발견 → 수정. codex 결과도 무비판 수용하지 않고 재검증 후 분류(5건 중 2 수정 / 3 기각).
- 수렴 기준: 신규 confirmed 0 라운드. R5 = 0, R6 = 0 → dry.
