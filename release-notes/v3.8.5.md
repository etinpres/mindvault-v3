# v3.8.5 — MEMORY.md 인덱스 200줄 hard limit 회피 (procedural/feedback 분리)

## Summary

Claude Code 는 메모리 인덱스 `MEMORY.md` 를 세션 시작 시 **첫 200줄 또는 25KB(먼저 도달)
만 로드하는 hard truncate** 다(공식: code.claude.com/docs/en/memory). 모든 메모리를 한 줄씩
나열하는 flat full index 는 이 한계에 필연적으로 부딪혀 반복적인 수동 슬림화를 강요했다.
회수(memory-recall)는 `MEMORY.md` 가 아니라 개별 `memory/*.md` 를 FTS5+vec hybrid 검색하므로,
인덱스 목차를 `MEMORY.md` 에서 빼도 회수에는 무영향이다.

→ procedural / feedback 전체 목차를 `MEMORY-PROCEDURAL.md` / `MEMORY-FEEDBACK.md` 로 분리한다
(둘은 SessionStart 미로드라 길이 제한 없음). `MEMORY.md` 본체는 project/user/reference + 포인터만
남아 한 슬롯 기준 199 → 65줄로 축소됐다. `close-session` §8 도 갱신해 향후 신규
procedural/feedback 인덱스가 분리 파일로 가도록 해 재발(반복 슬림화)을 영구 차단한다.

## Changed

- `skill/close-session.md` §8 — 신규 토픽 인덱스를 type 별 분기:
  procedural → `MEMORY-PROCEDURAL.md`, feedback → `MEMORY-FEEDBACK.md`,
  그 외(project/user/reference) → `MEMORY.md`
