# v3.9.0 — gbrain 차용: Contextual Retrieval + Eval Gate

## Summary

Garry Tan 의 [gbrain](https://github.com/garrytan/gbrain)(Postgres-native 14.6만 페이지 brain)과
MindVault v3 를 9단계 비교해 **베껴 명백히 이득인 2건**을 MV 핵심 원칙(CC 내부 전용·로컬 MLX
zero-cost·사용자 행동 제로·점진적 가치)을 하나도 깨지 않고 이식했다.

1. **Contextual Retrieval (CR)** — body 임베딩 전에 맥락 한 줄(name + synopsis)을 선붙여 별
   벡터(`embedding_ctx`)로 저장. gbrain 은 클라우드 Haiku 지만 MV 는 **로컬 Gemma 12B** 로
   synopsis 생성(zero-cost). 비용 전액 index-time — **회수 query-time 0ms 증가**.
2. **Eval Gate** — `query→정답 메모리(name)` 라벨 코퍼스(32 쿼리)로 **P@k/R@k/MRR** 산출,
   baseline 대비 회귀 시 **비영 exit**. MV 의 "측정 artifact"(66.3% vs 옛 2.6%/79% 혼란)를
   구조적으로 해결 — 검색 변경을 머지 전 "정확도 떨어뜨렸나"로 자동 판정.

## A/B 핵심 판정 (실측)

**CR 의 검색 개선 = 0.** title·synopsis 양 티어 A/B 모두 **완전 동률**(off=CR-on, Δ=0, 32쿼리
rank 변동 0). 이유: **MV 는 사람이 쓴 frontmatter `description` 을 별도 1.5x 가중 벡터로 이미
임베딩**해 맥락 신호를 내장 — gbrain(raw chunk만 임베딩)과 달리 CR 이 중복. 따라서:

- **CR 기본 비활성(off) 유지** — 신컬럼 미사용 시 회수 동작 **바이트 동일**(회귀 0). opt-in
  은 `MV3_CR_MODE=title|synopsis` 백필 + `MV3_CR_SEARCH=1`.
- **Eval Gate 의 1차 가치 = 회귀 보호** — off baseline 을 고정해 CR·미래 검색 변경의 정확도
  하락을 자동 차단.
